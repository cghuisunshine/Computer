Knight Tour Sample Code Explained

This code finds one knight's tour on a 5x5 chessboard, starting from the top-left corner (0, 0), and then prints the board after each move. A knight's tour means the knight moves in its usual L-shape, visits every square exactly once, and never lands on the same square twice.

Overall flow

  • Create an empty 5x5 board.
  • Start the knight at (0, 0).
  • Use recursion and backtracking to search for a full tour.
  • Use Warnsdorff's rule to try smarter moves first.
  • Store the successful route in path.
  • Replay the path and print the board after every move.

Why this works well

  • Backtracking lets the program undo failed paths.
  • Warnsdorff's rule prefers squares with fewer future choices first.
  • Together, these ideas make the search much faster than plain brute force.

Full Sample Code

This is the full sample code from the knight-tour question, shown in one place before the step-by-step explanation.

Python sample solution 
N = 5
moves = [
    (2, 1), (1, 2), (-1, 2), (-2, 1),
    (-2, -1), (-1, -2), (1, -2), (2, -1)
]

def print_grid(board):
    for row in board:
        print(" ".join(str(cell) for cell in row))
    print()

def is_valid(r, c, board):
    return 0 <= r < N and 0 <= c < N and board[r][c] == 0

def onward_count(r, c, board):
    count = 0
    for dr, dc in moves:
        nr, nc = r + dr, c + dc
        if is_valid(nr, nc, board):
            count += 1
    return count

def knight_tour(r, c, step, board, path):
    board[r][c] = step
    path.append((r, c))

    if step == N * N:
        return True

    next_moves = []
    for dr, dc in moves:
        nr, nc = r + dr, c + dc
        if is_valid(nr, nc, board):
            next_moves.append((onward_count(nr, nc, board), nr, nc))

    next_moves.sort()

    for _, nr, nc in next_moves:
        if knight_tour(nr, nc, step + 1, board, path):
            return True

    board[r][c] = 0
    path.pop()
    return False

search_grid = [[0] * N for _ in range(N)]
path = []
knight_tour(0, 0, 1, search_grid, path)

grid = [[0] * N for _ in range(N)]
step = 1
for r, c in path:
    grid[r][c] = step
    print_grid(grid)
    step += 1

1. Board Size and Knight Moves

Setup 
N = 5
moves = [
    (2, 1), (1, 2), (-1, 2), (-2, 1),
    (-2, -1), (-1, -2), (1, -2), (2, -1)
]
  • N = 5 means the board is 5 x 5.
  • moves stores all 8 possible knight moves.
  • For example, (2, 1) means move 2 rows down and 1 column right.
From a position (r, c), the program tries squares such as (r+2, c+1), (r+1, c+2), and the other 6 L-shaped choices.

2. Printing the Board

Printing function 
def print_grid(board):
    for row in board:
        print(" ".join(str(cell) for cell in row))
    print()
Example output 
[1, 0, 0, 0, 0]

becomes

1 0 0 0 0

This function prints the current board. The extra print() adds a blank line after the whole board so each step is easier to read.

3. Checking Whether a Move Is Allowed

Validation 
def is_valid(r, c, board):
    return 0 <= r < N and 0 <= c < N and board[r][c] == 0

A move is valid only if all three conditions are true:

  1. r is inside the board.
  2. c is inside the board.
  3. board[r][c] == 0, which means the square has not been visited yet.

This prevents the knight from going outside the board or revisiting a square.

4. Counting Future Choices from a Square

Warnsdorff helper 
def onward_count(r, c, board):
    count = 0
    for dr, dc in moves:
        nr, nc = r + dr, c + dc
        if is_valid(nr, nc, board):
            count += 1
    return count

This function counts how many valid next moves are available from (r, c).

The algorithm uses Warnsdorff's rule: try the move with the fewest onward moves first. That helps avoid getting stuck too early.

If one square has only 1 future exit and another has 5 future exits, it is usually smarter to visit the 1-exit square earlier.

5. Main Recursive Search

Start of the recursive function 
def knight_tour(r, c, step, board, path):
    board[r][c] = step
    path.append((r, c))

The parameters mean:

  • r, c: the current knight position
  • step: which move number this is
  • board: the board used for searching
  • path: the visited positions in order

These two lines mark the current square with the step number and save the position into path.

First call 
knight_tour(0, 0, 1, search_grid, path)

So the program starts by placing step 1 at (0, 0).

6. Base Case: All Squares Visited

Finish condition 
if step == N * N:
    return True

For a 5 x 5 board, N * N = 25. When step == 25, the knight has visited all 25 squares, so the tour is complete.

7. Build Candidate Next Moves

Collect possible next squares 
next_moves = []
for dr, dc in moves:
    nr, nc = r + dr, c + dc
    if is_valid(nr, nc, board):
        next_moves.append((onward_count(nr, nc, board), nr, nc))

This loop checks all 8 possible knight moves from the current square.

For every valid next square (nr, nc), it stores a tuple:

Stored data 
(onward_count(nr, nc, board), nr, nc)

So each candidate includes how many future moves it has, plus its row and column.

Example idea 
[(2, 2, 1), (4, 1, 2)]
  • Square (2, 1) has 2 onward moves.
  • Square (1, 2) has 4 onward moves.

8. Sort by Smallest Onward Count

Sorting step 
next_moves.sort()

This sorts the list in ascending order, so the algorithm tries the move with the smallest onward count first. That is the key heuristic.

9. Recursive Exploration

Try each candidate move 
for _, nr, nc in next_moves:
    if knight_tour(nr, nc, step + 1, board, path):
        return True

The program now tries each next move one by one. For each candidate square, it moves there, increases the step by 1, and continues searching recursively.

If any recursive call returns True, the program has found a complete tour, so this function also returns True.

10. Backtracking

Undo a failed path 
board[r][c] = 0
path.pop()
return False

If none of the next moves work, the algorithm must undo the current move. This is called backtracking.

  • Reset the square to 0 so it becomes unvisited again.
  • Remove the last position from path.
  • Return False so the previous call can try another move.

11. Preparing the Search

Set up the board and path 
search_grid = [[0] * N for _ in range(N)]
path = []
knight_tour(0, 0, 1, search_grid, path)
  • search_grid creates a 5 x 5 board filled with 0.
  • path starts empty.
  • The search begins at (0, 0) with step 1.

After this finishes, path contains the full successful route of the knight.

12. Replaying the Solution Step by Step

Replay loop 
grid = [[0] * N for _ in range(N)]
step = 1
for r, c in path:
    grid[r][c] = step
    print_grid(grid)
    step += 1

This part is separate from the search. search_grid was used to find the path, while grid is used to show the tour gradually.

  1. Place the current step number.
  2. Print the board.
  3. Increase the step.

So instead of printing only the final completed board, the program prints the board after move 1, move 2, move 3, and so on until move 25.

Final Summary

What path stores

After the search, path may look like [(0, 0), (2, 1), (4, 2), (3, 4), ...]. That means step 1 is at (0, 0), step 2 is at (2, 1), step 3 is at (4, 2), and so on.

Key ideas

The code combines recursive backtracking with Warnsdorff's rule. Backtracking fixes wrong choices, and the heuristic makes good choices earlier.
